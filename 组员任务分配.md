## 后端任务

### 1. 支付服务（payment-service）— 主要开发

当前 `PaymentController.java` 三个接口全是 TODO，需要全部实现。

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/payments` | POST | 发起支付，创建支付记录，模拟支付结果（80%成功/20%失败），支付完成后调用订单服务更新状态 |
| `/api/payments/{orderId}` | GET | 根据 orderId 查询支付状态 |
| `/api/payments/health` | GET | 健康检查，返回 OK |

**支付服务还需新增**：
- `PaymentService.java` — 业务逻辑层（参考 order-service 的 OrderService）
- 支付完成后通过 HTTP 调用订单服务的状态更新接口

**涉及文件**：
- `payment-service/src/main/java/com/minipay/payment/controller/PaymentController.java` — 填充业务逻辑
- `payment-service/src/main/java/com/minipay/payment/service/PaymentService.java` — 新建

### 2. 订单服务（order-service）— 补充接口

订单服务已有创建、查询、健康检查，但**缺少订单状态更新接口**，支付服务回调时需要。

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/orders/{orderId}/status` | PUT | 更新订单状态（PENDING → PAID/FAILED），供支付服务回调 |

**涉及文件**：
- `order-service/src/main/java/com/minipay/order/controller/OrderController.java` — 新增接口
- `order-service/src/main/java/com/minipay/order/service/OrderService.java` — 新增方法

### 3. 接口请求/响应格式

```
POST /api/orders
请求：{ "amount": 100, "memberId": 1 }
响应：{ "code": 200, "message": "创建成功", "data": { "orderId": "xxx", "amount": 100.00, "status": "PENDING", ... }, "success": true }

GET /api/orders/{orderId}
响应：{ "code": 200, "message": "查询成功", "data": { "orderId": "xxx", "status": "PAID", ... }, "success": true }

PUT /api/orders/{orderId}/status
请求：{ "status": "PAID" }
响应：{ "code": 200, "message": "更新成功", "data": { "orderId": "xxx", "status": "PAID" }, "success": true }

POST /api/payments
请求：{ "orderId": "xxx", "amount": 100.00 }
响应：{ "paymentId": "P001", "orderId": "xxx", "status": "SUCCESS", "paidAt": "..." }

GET /api/payments/{orderId}
响应：{ "paymentId": "P001", "orderId": "xxx", "status": "SUCCESS", "amount": 100.00, "paidAt": "..." }
```

### 4. 服务间通信

支付完成后，支付服务通过 HTTP 调用订单服务更新订单状态：

```
PUT http://order-service:8081/api/orders/{orderId}/status
请求：{ "status": "PAID" }  或  { "status": "FAILED" }
```

### 5. 现有基础设施

| 组件 | 说明 |
|------|------|
| CommonResp | 统一响应格式 `{ code, message, data, success }` |
| OrderReq | 创建订单请求体 `{ memberId, amount }` |
| LogInterceptor | 日志拦截器，自动生成 LOG_ID |
| LoginFilter | 网关登录过滤器，校验 JWT token |
| JwtUtil | JWT 工具类，生成和验证 token |
| Nacos | 远程服务发现 nacos.linqin.space |
| 数据库 | 订单服务已配置远程 MySQL（8.137.58.214），支付服务需同步配置 |

## 前端任务

### 1. 基础设施

| 任务 | 说明 |
|------|------|
| 安装 vue-router | `npm install vue-router@4` |
| 配置路由 | 新建 `router/index.js`，配置3个页面路由 |
| 注册 router | 在 `main.js` 中引入并注册 |

| API 封装 | 完善 `api/index.js`，封装所有后端接口调用 |

### 2. 页面

| 页面 | 路由 | 功能 |
|------|------|------|
| 创建订单 | `/` | 输入金额 + 描述，点击创建，跳转到支付页 |
| 发起支付 | `/pay/:orderId` | 展示订单信息，点击支付按钮，显示支付结果 |
| 结果查询 | `/query` | 输入 orderId，查询并展示订单状态和支付状态 |

### 3. API 封装约定

```javascript
// api/index.js
import axios from 'axios'

const api = axios.create({
  baseURL: '/api'
})

export const createOrder = (data) => api.post('/orders', data)
export const getOrder = (orderId) => api.get(`/orders/${orderId}`)
export const updateOrderStatus = (orderId, status) => api.put(`/orders/${orderId}/status`, { status })
export const createPayment = (data) => api.post('/payments', data)
export const getPayment = (orderId) => api.get(`/payments/${orderId}`)
```
